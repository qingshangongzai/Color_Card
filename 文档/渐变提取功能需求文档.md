# 渐变提取功能需求文档

## 1. 功能概述

### 1.1 功能名称
渐变提取

### 1.2 功能定位
通过选择起始色和结束色，生成中间的渐变色序列，为用户提供便捷的渐变色方案生成工具。

### 1.3 导航位置
位于"明度提取"之后，"配色生成"之前。

---

## 2. 界面布局说明

### 2.1 整体结构

界面采用上下分区布局：
- **上半部分**：控制区 + 渐变预览区
- **中间**：操作按钮区
- **下半部分**：色卡排列区

### 2.2 上半部分布局

#### 左侧：控制区
- **起始颜色选择**
  - 颜色圆点（点击弹出颜色选择器）
  - 标签："起始颜色"
  - HEX输入框（复用配色管理面板的16进制输入框样式）
  
- **结束颜色选择**
  - 颜色圆点（点击弹出颜色选择器）
  - 标签："结束颜色"
  - HEX输入框（复用配色管理面板的16进制输入框样式）

- **中间色数量控制**
  - 标签："中间色数量"
  - Slider滑块控件
  - 数值显示（1-10）

#### 右侧：渐变预览区
- **渐变显示区域**
  - 宽度：根据布局自适应
  - 显示方式：将渐变色分成一个个竖条
  - 列宽：由颜色总数决定（总宽度 / 颜色数量）
  - 实时更新：颜色或步数改变时立即重绘

### 2.3 中间操作区

两个操作按钮横向排列：
- **随机按钮**
  - 图标：刷新/随机图标
  - 文本："随机"
  - 功能：随机生成起始颜色和结束颜色
  
- **收藏按钮**
  - 图标：心形图标
  - 文本："收藏"
  - 功能：将当前渐变色方案添加到收藏（复用其他界面的收藏功能）

### 2.4 下半部分：色卡排列区

**布局策略**：参考"内置色彩"和"配色管理"面板的色卡布局

| 颜色数量 | 布局方式 |
|---------|---------|
| ≤5个 | 一行显示所有颜色 |
| 6个 | 一行显示6个颜色 |
| 7-10个 | 第一行5个，第二行剩余 |
| >10个 | 每行最多5个，多行排列 |

**色卡组件**：复用现有的色卡组件，显示：
- 颜色块
- HEX值
- 配置的色彩模式值（HSB/LAB等）
- 复制按钮

### 2.5 组件复用说明

| 组件 | 复用来源 |
|-----|---------|
| HEX输入框 | 配色管理面板的16进制输入框 |
| 颜色选择器 | 现有的ColorPickerDialog |
| 色卡组件 | 内置色彩/配色管理面板的色卡 |
| 收藏功能 | 其他界面的收藏实现 |
| Slider控件 | qfluentwidgets的Slider |

## 3. 核心功能需求

### 3.1 颜色选择
- 支持选择起始颜色
- 支持选择结束颜色
- 支持通过颜色选择器或HEX值输入

### 3.2 步数控制
- 控制中间色的数量
- 范围：1-10步
- 默认：2步（生成4个颜色：起始色 + 2个中间色 + 结束色）

### 3.3 渐变预览
- 实时显示生成的渐变色序列
- 右侧预览区显示渐变色竖条
- 下方色卡区显示每个颜色的详细信息

### 3.4 操作功能
- 随机生成：随机生成起始色和结束色
- 收藏：将当前渐变色方案添加到收藏

## 4. 渐变颜色空间设置

### 4.1 设置项说明

| 属性 | 值 |
|-----|-----|
| 设置键 | `settings.gradient_color_space` |
| 默认值 | `lab` |
| 可选值 | `rgb`, `hsb`, `lab` |

### 4.2 三种模式说明

#### RGB空间
- **技术特点**：直接对R、G、B三个通道进行线性插值
- **适用场景**：技术用户，需要精确控制颜色数值
- **优点**：计算简单，结果可预测
- **缺点**：某些颜色过渡可能出现不自然的灰色

#### HSB空间
- **技术特点**：色相沿最短路径过渡，饱和度和明度线性插值
- **适用场景**：设计场景，注重色相的自然过渡
- **优点**：色相环过渡自然，符合设计直觉
- **缺点**：需要处理色相跨越0°/360°的情况
- **特殊处理**：色相选择最短路径插值

#### LAB空间（推荐）
- **技术特点**：在感知均匀的颜色空间中进行插值
- **适用场景**：追求最佳视觉效果
- **优点**：人眼感知最平滑的过渡
- **缺点**：计算相对复杂

### 4.3 设置界面集成
- 设置分组："渐变提取"
- 设置项名称：渐变颜色空间
- 设置项描述：选择渐变色插值的颜色空间
- 选项显示格式："模式名 - 简短说明"

## 5. 核心算法需求

### 5.1 算法输入
- 起始颜色HEX值
- 结束颜色HEX值
- 中间色数量（步数）
- 颜色空间模式

### 5.2 算法输出
- RGB颜色列表，包含起始色、所有中间色、结束色

### 5.3 算法流程

1. **解析HEX值**：将起始色和结束色的HEX值转换为RGB元组

2. **颜色空间转换**：
   - RGB模式：保持RGB值不变
   - HSB模式：将RGB转换为HSB
   - LAB模式：将RGB转换为LAB

3. **色相归一化处理**（仅HSB模式）：
   - 检测色相差值是否超过180°
   - 如超过，调整色相值使插值沿最短路径

4. **线性插值**：
   - 根据步数计算插值点
   - 在选定颜色空间进行线性插值

5. **转换回RGB**：
   - HSB/LAB模式需将结果转换回RGB

6. **返回结果**：返回完整的RGB颜色列表

### 5.4 边界情况处理

- **色相跨越0°/360°**：HSB模式下选择最短路径插值
- **颜色空间转换精度**：确保转换后的RGB值在有效范围内(0-255)
- **步数为0**：至少返回起始色和结束色

## 6. 配置存储

### 6.1 配置文件结构

```json
{
    "settings": {
        "gradient_color_space": "lab"
    }
}
```

### 6.2 配置管理
- 使用现有的配置管理器（ConfigManager）
- 支持配置的读取、写入和默认值回退
- 配置变更时触发信号通知

### 7.1 新增信号

| 信号名 | 参数 | 说明 |
|-------|------|------|
| `gradient_color_space_changed` | `str` | 渐变颜色空间设置改变时发射 |

### 7.2 信号连接
- SettingsInterface发射信号
- GradientExtractInterface接收信号并更新当前使用的颜色空间

## 10. 技术约束

### 10.1 代码规范
- 遵循PEP 8代码风格
- 使用4空格缩进
- 行长度不超过100字符
- 所有公共类和方法必须添加中文文档字符串

### 10.2 架构约束
- UI层只负责UI展示，功能逻辑下沉到业务层
- 使用现有的颜色转换工具函数
- 复用现有的颜色选择器和色卡预览组件
- 遵循现有的信号同步防循环机制

### 10.3 性能约束
- 颜色计算响应时间 < 100ms
- 使用缓存避免重复计算
