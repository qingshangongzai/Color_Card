## 一、问题概述

### 1.1 问题统计

| 问题等级 | 数量 | 主要问题 |
|----------|------|----------|
| 🔴 高风险 | 5 | `terminate()` 使用、信号连接类型、QImage 引用、**信号未断开**、**延迟加载 parent 问题** |
| 🟡 中风险 | 5 | 清理时序、父对象设置、UI 阻塞、InfoBar parent、QTimer 延迟回调 |
| 🟢 低风险 | 1 | 异常处理过于宽泛 |

### 1.2 闪退问题定位

#### 配色管理面板闪退原因
1. **HistogramCalculator 的 `terminate()` 调用** - 切换分组或重新加载时强制终止导致崩溃
2. **信号连接类型问题** - 异步加载线程信号未使用 `QueuedConnection`，跨线程 UI 更新
3. **QImage 引用问题** - `LuminanceCalculator` 直接引用 QImage，计算过程中图片被释放

#### 配色生成面板收藏闪退原因
1. **主要原因**：`EditPaletteDialog` 和 `ColorInputRow` 中的 `qconfig.themeChangedFinished` 信号未断开连接
2. **次要原因**：InfoBar parent 可能无效、QTimer 延迟回调问题、直接引用颜色数据

#### 延迟加载服务导致的潜在问题
1. **parent 对象生命周期问题** - 首次创建服务时传入的 parent 可能已被销毁
2. **信号连接时序问题** - `_setup_service_connections()` 在首次获取服务时才执行
3. **线程安全问题** - parent 参数在不同调用间可能不一致

### 3.3 中优先级修复（P2）

#### 修复 3.1: 移除 HistogramCalculator 的 terminate() 调用

**文件**: `core/histogram_service.py`

**位置**: 第384-414行

**当前代码**:
```python
def _safe_stop_calculator(self, calculator: Optional[HistogramCalculator]) -> None:
    if calculator is None:
        return

    # 断开所有信号连接
    try:
        calculator.finished.disconnect()
    except (TypeError, RuntimeError):
        pass
    try:
        calculator.error.disconnect()
    except (TypeError, RuntimeError):
        pass

    # 请求取消
    calculator.cancel()

    # 等待线程结束
    if calculator.isRunning():
        calculator.wait(200)
    
    # 如果线程仍在运行，强制终止（不推荐，但必要时使用）
    if calculator.isRunning():
        calculator.terminate()
        calculator.wait(100)
```

**修复代码**:
```python
def _safe_stop_calculator(self, calculator: Optional[HistogramCalculator]) -> None:
    """安全停止直方图计算器（不使用 terminate()）"""
    if calculator is None:
        return

    # 断开所有信号连接
    try:
        calculator.finished.disconnect()
    except (TypeError, RuntimeError):
        pass
    try:
        calculator.error.disconnect()
    except (TypeError, RuntimeError):
        pass

    # 请求取消
    calculator.cancel()

    # 等待线程结束（增加等待时间）
    if calculator.isRunning():
        calculator.wait(500)  # 增加到500ms
    
    # 删除：不要强制终止
    # 如果线程仍在运行，让它在后台自然结束
    # 下次创建新计算器时会替换旧实例
```

#### 修复 3.6: 添加服务类析构函数（紧急修复）

**问题**: 当服务类（如 `HistogramService`）被销毁时，其中的线程可能还在运行，导致 `QThread: Destroyed while thread is still running` 错误。

**解决方案**: 为所有服务类添加 `__del__` 析构函数，确保在对象销毁前停止所有线程。

**修复文件**:
- `core/histogram_service.py`
- `core/luminance_service.py`
- `core/color_service.py`
- `core/palette_service.py`
- `core/image_service.py`

**修复代码示例**:
```python
def __del__(self):
    """析构函数：确保所有线程在对象销毁前停止"""
    self.cancel_all()
    # 等待所有线程完全结束
    for calculator in [self._luminance_calculator, self._rgb_calculator, self._hue_calculator]:
        if calculator is not None and calculator.isRunning():
            calculator.wait(1000)  # 等待最多1秒
```

#### 修复 4.1: 优化 cancel() 方法避免阻塞 UI

**所有线程类的 cancel() 方法**:

```python
def cancel(self) -> None:
    """取消任务（异步，不阻塞调用线程）"""
    self._is_cancelled = True
    # 不在主线程中调用 wait()，避免阻塞 UI
    # 线程会在检查 _is_cancelled 标志后自然退出
```

#### 修复 4.2: 添加线程安全文档注释

**所有线程类都应添加文档注释**:

```python
class HistogramCalculator(QThread):
    """
    直方图计算线程
    
    线程安全说明：
    - 所有信号都使用 QueuedConnection，确保跨线程安全
    - QImage 数据在构造时复制，避免主线程修改
    - cancel() 方法设置标志位，不阻塞调用线程
    - 不使用 terminate()，通过标志位优雅退出
    
    使用示例：
        calculator = HistogramCalculator(image, parent=self)
        calculator.finished.connect(self._on_finished, Qt.ConnectionType.QueuedConnection)
        calculator.error.connect(self._on_error, Qt.ConnectionType.QueuedConnection)
        calculator.start()
    """
```

### 7.1 技术风险

| 风险项 | 风险等级 | 应对措施 |
|--------|----------|----------|
| 移除 terminate() 后线程无法退出 | 低 | 增加等待时间到 500ms，添加超时日志 |

| 延迟加载服务初始化失败 | 中 | 添加异常处理和错误提示 |
| 信号断开时序问题 | 中 | 使用 try-except 捕获异常 |

| 风险项 | 风险等级 | 应对措施 |
|--------|----------|----------|
| 不同操作系统表现不一致 | 低 | 在 Windows/macOS/Linux 上测试 |

## 九、后续优化建议

### 9.1 短期优化（1-3个月）

1. **引入线程池**: 使用 `QThreadPool` 管理线程，避免频繁创建销毁
2. **任务队列**: 实现任务队列，避免并发任务过多
3. **性能监控**: 添加线程性能监控，及时发现瓶颈

### 9.2 中期优化（3-6个月）

1. **异步框架**: 考虑引入异步框架（如 `asyncio`）简化异步逻辑
2. **线程安全容器**: 使用线程安全的数据结构，减少锁竞争
3. **智能调度**: 根据任务类型智能选择线程池

### 9.3 长期优化（6-12个月）

1. **Python 3.14t 适配**: 利用 free-threading 特性优化并行计算
2. **多进程架构**: 将 CPU 密集型任务移到独立进程
3. **分布式计算**: 支持分布式计算，处理大规模数据
