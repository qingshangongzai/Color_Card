# Color Card 日志系统创建方案

## 1. 概述

### 1.1 设计目标

建立统一的日志系统，用于：

- **问题排查**：用户反馈问题时，通过日志快速定位错误位置和上下文
- **行为追踪**：记录用户关键操作路径，帮助理解难以复现的 Bug
- **性能监控**：记录耗时操作，发现性能瓶颈
- **运行监控**：记录程序启动、配置加载、异常等关键事件

### 1.2 设计原则

遵循项目核心开发原则：

- **简洁实用**：不过度设计，满足核心需求即可
- **渐进集成**：逐步添加到关键模块，不一次性全量改造
- **零配置**：无需用户设置，默认自动启用
- **开发友好**：日志格式清晰，便于开发人员阅读分析

## 3. 日志配置规范

### 3.1 日志级别定义

| 级别 | 用途 | 示例 |
|:---:|:---|:---|
| `DEBUG` | 开发调试信息 | 函数调用、变量值、内部状态 |
| `INFO` | 关键操作记录 | 打开图片、提取颜色、保存配置 |
| `WARNING` | 非致命异常 | 文件不存在但可恢复、配置项缺失使用默认值 |
| `ERROR` | 需要关注的问题 | 保存失败、解析错误、服务初始化失败 |
| `CRITICAL` | 致命错误 | 程序无法继续运行 |

### 3.2 日志格式

```
[时间] [级别] [模块名] 消息内容
```

示例：
```
[2026-02-28 14:30:25] [INFO] [color_service] 提取主色调: 图片=photo.jpg, 颜色数量=5
[2026-02-28 14:30:26] [WARNING] [config] 配置项缺失，使用默认值: key=settings.theme, default=auto
[2026-02-28 14:30:30] [ERROR] [image_service] 图片加载失败: path=invalid.jpg, error=文件不存在
```

### 3.3 日志存储规范

**存储位置**：
- 日志存储在配置目录下的 `logs/` 子目录
- 与配置文件同一目录：`~/.color_card/logs/`

**文件命名**：
- 当前日志：`color_card.log`
- 历史日志：`color_card_YYYYMMDD.log`

**轮转策略**：
- 按日期轮转：每天创建新日志文件
- 保留数量：保留最近 7 天的日志
- 文件大小限制：单个日志文件最大 10MB

## 4. 日志内容规范

### 4.1 必须记录的场景

**系统事件**：
- 程序启动/退出
- 配置加载/保存
- 主题切换
- 语言切换

**用户操作**：
- 打开/关闭图片
- 提取颜色（色彩/明度/渐变）
- 生成配色方案
- 保存/删除收藏
- 导入/导出配色
- 场景模板操作

**错误和异常**：
- 文件操作失败
- 图片解析错误
- 配置读写错误
- 服务初始化失败

### 4.2 用户操作日志格式

使用结构化格式记录用户操作，便于分析：

```python
log_user_action(
    action="open_image",
    params={"path": "photo.jpg", "size": "1920x1080"},
    result="success"
)
```

输出示例：
```
[2026-02-28 14:30:25] [INFO] [user_action] open_image: path=photo.jpg, size=1920x1080, result=success
```

### 4.3 性能日志格式

记录关键操作的耗时：

```python
with log_performance("generate_scheme", {"algorithm": "monochromatic"}):
    # 执行配色生成
    pass
```

输出示例：
```
[2026-02-28 14:30:25] [INFO] [performance] generate_scheme: algorithm=monochromatic, elapsed=45ms
```

## 5. 实现规范

### 5.1 代码组织

**文件**：`core/logger.py`

```python
# 标准库导入
import logging
import logging.handlers
import os
import sys
from datetime import datetime
from pathlib import Path
from typing import Any, Dict, Optional, Tuple
from contextlib import contextmanager
import time


class LoggerManager:
    """日志管理器，统一管理应用程序日志配置"""

    CONFIG_DIR_NAME: str = ".color_card"
    LOG_DIR_NAME: str = "logs"
    LOG_FILE_NAME: str = "color_card.log"
    MAX_BYTES: int = 10 * 1024 * 1024  # 10MB
    BACKUP_COUNT: int = 7

    def __init__(self) -> None:
        """初始化日志管理器"""
        self._log_dir: Path = self._get_log_dir()
        self._logger: Optional[logging.Logger] = None
        self._initialized: bool = False

    def _get_log_dir(self) -> Path:
        """获取日志目录路径

        Returns:
            Path: 日志目录的完整路径
        """
        home_dir = Path.home()
        config_dir = home_dir / self.CONFIG_DIR_NAME
        return config_dir / self.LOG_DIR_NAME

    def _ensure_log_dir(self) -> None:
        """确保日志目录存在"""
        self._log_dir.mkdir(parents=True, exist_ok=True)

    def initialize(self, level: int = logging.INFO) -> None:
        """初始化日志系统

        Args:
            level: 日志级别，默认为 INFO
        """
        if self._initialized:
            return

        self._ensure_log_dir()

        # 创建根日志记录器
        self._logger = logging.getLogger("color_card")
        self._logger.setLevel(level)

        # 清除现有处理器
        self._logger.handlers.clear()

        # 文件处理器（按大小轮转）
        log_path = self._log_dir / self.LOG_FILE_NAME
        file_handler = logging.handlers.RotatingFileHandler(
            log_path,
            maxBytes=self.MAX_BYTES,
            backupCount=self.BACKUP_COUNT,
            encoding="utf-8"
        )
        file_handler.setLevel(level)

        # 控制台处理器
        console_handler = logging.StreamHandler(sys.stdout)
        console_handler.setLevel(level)

        # 格式化器
        formatter = logging.Formatter(
            "[%(asctime)s] [%(levelname)s] [%(name)s] %(message)s",
            datefmt="%Y-%m-%d %H:%M:%S"
        )
        file_handler.setFormatter(formatter)
        console_handler.setFormatter(formatter)

        self._logger.addHandler(file_handler)
        self._logger.addHandler(console_handler)

        self._initialized = True
        self._logger.info("日志系统初始化完成")

    def get_logger(self, name: str) -> logging.Logger:
        """获取模块级日志记录器

        Args:
            name: 模块名称

        Returns:
            logging.Logger: 配置好的日志记录器
        """
        if not self._initialized:
            self.initialize()
        return logging.getLogger(f"color_card.{name}")

    def get_log_path(self) -> Path:
        """获取当前日志文件路径

        Returns:
            Path: 日志文件路径
        """
        return self._log_dir / self.LOG_FILE_NAME

    def get_log_dir(self) -> Path:
        """获取日志目录路径

        Returns:
            Path: 日志目录路径
        """
        return self._log_dir


# 全局日志管理器实例
_logger_manager: Optional[LoggerManager] = None


def get_logger_manager() -> LoggerManager:
    """获取全局日志管理器实例

    Returns:
        LoggerManager: 日志管理器实例
    """
    global _logger_manager
    if _logger_manager is None:
        _logger_manager = LoggerManager()
    return _logger_manager


def get_logger(name: str) -> logging.Logger:
    """获取模块级日志记录器

    Args:
        name: 模块名称

    Returns:
        logging.Logger: 配置好的日志记录器
    """
    return get_logger_manager().get_logger(name)


def log_user_action(action: str, params: Optional[Dict[str, Any]] = None, result: str = "") -> None:
    """记录用户操作

    Args:
        action: 操作名称
        params: 操作参数
        result: 操作结果
    """
    logger = get_logger("user_action")
    param_str = ", ".join(f"{k}={v}" for k, v in (params or {}).items())
    result_str = f", result={result}" if result else ""
    logger.info(f"{action}: {param_str}{result_str}")


@contextmanager
def log_performance(operation: str, params: Optional[Dict[str, Any]] = None):
    """性能日志上下文管理器

    Args:
        operation: 操作名称
        params: 操作参数

    用法:
        with log_performance("generate_scheme", {"algorithm": "mono"}):
            # 执行操作
            pass
    """
    logger = get_logger("performance")
    start_time = time.time()

    try:
        yield
    finally:
        elapsed = int((time.time() - start_time) * 1000)  # 转换为毫秒
        param_str = ", ".join(f"{k}={v}" for k, v in (params or {}).items())
        param_prefix = f"{param_str}, " if param_str else ""
        logger.info(f"{operation}: {param_prefix}elapsed={elapsed}ms")
```

### 5.2 使用示例

**模块级日志记录**：

```python
# core/color_service.py
from core.logger import get_logger

# 创建模块级日志记录器
logger = get_logger("color_service")

class ColorService:
    def extract_colors(self, image_path: str, count: int = 5):
        """提取主色调"""
        logger.info(f"开始提取主色调: path={image_path}, count={count}")

        try:
            # 执行提取逻辑
            colors = self._do_extract(image_path, count)
            logger.info(f"主色调提取完成: 提取到 {len(colors)} 个颜色")
            return colors
        except Exception as e:
            logger.error(f"主色调提取失败: error={str(e)}")
            raise
```

**用户操作记录**：

```python
# ui/color_extract.py
from core.logger import log_user_action

class ColorExtractInterface(QWidget):

        """打开图片"""
        file_path = self._show_file_dialog()
        if file_path:
            log_user_action(
                action="open_image",
                params={"path": file_path, "source": "color_extract"},
                result="success"

            self._load_image(file_path)
```

## 6. 集成规范

### 6.1 初始化时机

在 `main.py` 中尽早初始化日志系统：

```python
# main.py
def main():
    # 初始化日志系统（在创建 QApplication 之前）
    from core.logger import get_logger_manager
    logger_manager = get_logger_manager()
    logger_manager.initialize()

    logger = get_logger("main")
    logger.info("应用程序启动")

    app = QApplication(sys.argv)
    # ... 其余初始化代码
```

### 6.2 异常捕获

在 `main.py` 中添加全局异常处理：

```python
# main.py
def main():
    # ... 初始化代码

    logger = get_logger("main")

    try:
        sys.exit(app.exec())
    except KeyboardInterrupt:
        logger.info("程序被用户中断 (Ctrl+C)")
        sys.exit(0)
    except Exception as e:
        logger.critical(f"程序发生未捕获异常: {str(e)}", exc_info=True)
        raise
```

## 8. 注意事项

### 8.1 性能考虑

- 日志写入使用异步方式，不阻塞主线程
- 生产环境默认使用 INFO 级别，避免 DEBUG 日志过多
- 日志文件大小限制和轮转，防止占用过多磁盘空间

### 8.2 隐私保护

- 日志中不记录敏感信息（如用户文件内容的详细数据）
- 图片路径可以记录，但避免记录图片像素数据
- 所有数据存储在本地，除非用户主动上传，否则不会外传

### 8.3 与现有代码的兼容

- 保持与现有 `print()` 调试语句共存，逐步替换
- 不强制要求所有模块立即接入日志系统
- 遵循渐进式集成原则
