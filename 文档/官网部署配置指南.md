## 部署方式

### 方式一：docs 文件夹部署（推荐）

GitHub Pages 支持直接从 `docs` 文件夹部署静态网站。

#### 配置步骤

1. **确保文件已放入 docs 文件夹**
   - `docs/index.html` - 官网主页
   - `docs/logo.png` - Logo 图片
   - `docs/palettes/` - 配色数据文件夹

2. **提交代码到 GitHub**
   ```bash
   git add docs/
   git add .github/workflows/pages.yml
   git commit -m "添加官网页面到 docs 文件夹"
   git push origin main
   ```

3. **启用 GitHub Pages**
   - 打开 GitHub 仓库页面
   - 点击 **Settings** → **Pages**
   - **Source** 选择 **GitHub Actions**
   - 等待 2-3 分钟

4. **访问网站**
   ```
   https://你的用户名.github.io/仓库名/
   ```
   例如：`https://qingshangongzai.github.io/Color_Card/`

---

### 方式二：独立仓库部署

将官网放在独立的 GitHub 仓库中。

#### 配置步骤

1. **创建新仓库**
   - 仓库名：`color-card-website`
   - 设置为 Public

2. **上传文件**
   ```bash
   cd website
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/你的用户名/color-card-website.git
   git branch -M main
   git push -u origin main
   ```

3. **启用 GitHub Pages**
   - Settings → Pages → Source → GitHub Actions

4. **访问网站**
   ```
   https://你的用户名.github.io/color-card-website/
   ```

## 自动部署配置

### GitHub Actions 工作流文件

文件路径：`.github/workflows/pages.yml`

```yaml
name: Deploy GitHub Pages

on:
  push:
    branches: [ main, master ]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Pages
        uses: actions/configure-pages@v4
        with:
          enablement: true

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: './docs'

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

## 更新网站

### 修改网站内容

1. 编辑 `website/index.html` 文件
2. 测试修改效果（本地运行）
3. 将修改同步到 `docs/index.html`

### 同步命令

```bash
# 从开发目录同步到部署目录
copy website\index.html docs\index.html

# 提交到 GitHub
git add docs/
git commit -m "更新官网内容"
git push
```

GitHub Actions 会自动重新部署网站。

## 自定义域名（可选）

### 绑定自定义域名

1. 在 `docs` 文件夹中创建 `CNAME` 文件
2. 文件内容填写你的域名，例如：
   ```
   colorcard.example.com
   ```

3. 提交到 GitHub
   ```bash
   git add docs/CNAME
   git commit -m "添加自定义域名"
   git push
   ```

4. 在你的域名服务商处添加 DNS 记录：
   - 类型：CNAME
   - 主机：colorcard
   - 值：你的用户名.github.io

## 常见问题

### Q1: 部署后页面显示空白？

**原因**：通常是资源路径问题

**解决**：
- 检查 `index.html` 中的路径是否为相对路径（如 `./logo.png`）
- 检查浏览器控制台是否有 404 错误

### Q2: 配色数据没有显示？

**原因**：palettes 文件夹没有正确上传

**解决**：
```bash
# 检查文件是否存在
ls docs/palettes/

# 重新复制
copy website\public\palettes\* docs\palettes\
git add docs/palettes/
git push
```

### Q3: 如何查看部署日志？

**方法**：
1. 打开 GitHub 仓库
2. 点击 **Actions** 标签
3. 查看最新的工作流运行记录
4. 点击失败的步骤查看详细日志

### Q4: 部署成功后多久能访问？

**答**：通常 1-5 分钟。首次部署可能需要更长时间。

### Q5: 可以同时使用 Gitee Pages 吗？

**答**：可以！建议同时部署到 GitHub 和 Gitee：
- GitHub：国际访问
- Gitee：国内访问更快
